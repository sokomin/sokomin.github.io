パッケージ開封シミュレーター

各ボタンで 100 個分の各種アイテムを開封し、結果を所持数に加算します。

  • 最下級（100個） N：90% / R：10%
  • 下級（100個） N：75% / R：20% / HR：5%
  • 鱗（100個） N：52% / R：26% / HR：13% / SR：7% / LR：2%
※各アイテムの等級入手確率値はシミュレーター用の暫定値です。実際の確率とは異なります。
　最新の確率情報は公式サイトをご確認ください。



シミュレーターの仕様

（１）1回の合成で２つを１つにできます。
（２）下の等級のアイテムから順番に片づけます
（３）もし目当ての等級以上のアイテムが発生したら合成せずそのままとします。
（４）逆に要求を満たすまで減り続けて最終的にアイテムの所持数が7個になったら終了です。
　※最初から7個未満の場合は何もせず終了とします。
　※N〜LR のどの等級も 2 個以上なくて、もう合成できない状態になったら終了



期待値関連

N からスタートして その等級を初めて1個作るまでに必要な N 個数の期待値は以下です。
3列目は1つ上の等級を作成するのに必要な個数の期待値となります。

目標等級 必要な N の期待値 1つ上を作成する期待値
R 約 2.70 個 約 2.70 個（N）
HR 約 10.45 個 約 5.10 個（R）
SR 約 100.74 個 約 13.33 個（HR）
LR 約 1,721個 約 22.22 個（SR）
GR 約 128,671個 約 88.89 個（LR）

「R」は N だけをひたすら突っ込み続けて、初めて R が1個できるまでの期待値が 約 2.7 個
「HR」は N だけを供給して、初めて HR が1個できるまでに必要な N が 約 10.4 個
…という感じで、上に行くほど爆発的に N コストが増えていき、
最終的に GR 1個には 約 12.9万個の N が必要、という結果になっています。
